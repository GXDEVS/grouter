Como o grouter funciona

Serviço local em background. Qualquer provedor. Seus dados nunca saem da máquina.

Rotação de Contas

fill-first esgota a conta principal antes de avançar para a próxima.

round-robin distribui requisições entre contas com stickiness por sessão configurável.

Zero MITM · Apenas Local

Sem certificados. Sem interceptação.

O tráfego passa por localhost:3099 e os dados ficam em ~/.grouter/grouter.db.

Login Automático (OAuth)

Você faz login uma vez no browser.

O grouter salva o token, renova quando expira e gerencia o fluxo de redirect sozinho, sem intervenção manual.

TERMINAL
# 1. adiciona uma conexão — guia pelo OAuth
$ grouter add
✔ Device code received
Authorize: https://claude.ai/authorize?user_code=...
✔ Authorization successful!
✔ Account saved (priority 1)
# 2. inicia o daemon proxy
$ grouter serve on
● Proxy started   PID 80708   →  http://localhost:3099
stop:  grouter serve off
logs:  grouter serve logs
# 3. conecta sua ferramenta ao proxy
$ grouter up openclaude
✔ ~/.openclaude/settings.json   → env block written
✔ PowerShell profile             → updated
endpoint   http://localhost:3099/v1
